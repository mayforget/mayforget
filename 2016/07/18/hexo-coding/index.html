Mac上使用hexo+coding上搭建个人博客

发表于 2016-07-18   |     |   阅读次数

『Hexo是一个快速、简洁且高效的博客框架。Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。』——简介来自官网

之前折腾了几年WordPress和Typecho，最近接触了Jekyll、Hexo之后觉得单纯的写点什么还是这个好，最后选择了Hexo，搭建Hexo的平台有很多，比如Github、Coding等，但是出于国内访问速度考虑我选择了Coding，所以也就有了今天的Hexo+Coding博客搭建笔记。我这hexo版本是3.2.2，版本不同配置可能有些不同。

环境搭建

1、Node.js的安装

Mac直接官网下载安装，点击官网打开链接找到pkg文件下载进行安装

2、Git的安装

本地内容提交需要Git。如果你安装了Xcode那就自带Git，你可以使用以下命令来查看本机是否安装

git --version

安装Hexo

3、本地安装Hexo

执行如下npm命令安装hexo

sudo npm install -g hexo

4、初始化

安装完成后，新建一个博客目录，切换到该目录下执行命令

cd 博客文件夹路径
hexo init

5、安装Server组件

继续在该目录下执行命令

sudo npm install hexo-server --save

启动本地服务

6、生成静态页面

在博客目录下执行以下命令生成静态页面

hexo generate （或者hexo g）

7、启动本地服务

预览调试，博客目录下执行如下命令启动本地服务器，成功后可看到这个链接 http://localhost:4000 复制到浏览器中进行预览。

hexo server （或者hexo s）

成功后界面如下图：

配置Coding

8、Coding上新建项目

在Coding上新建一个项目，然后配置SSH公钥，具体参考官方文档

9、修改本地配置文件

修改本地博客目录下的 config.yml 文件，具体可参考官方文档，主要修改参数如下

deploy:
type: git       #部署方式，这里是git
repo:           #Coding地址，例如：git@git.coding.net:mayforget/blog.git   
branch:         #分支名，例如：master

部署

10、安装deploy组件

在博客目录下执行一下命令

sudo npm install hexo-deployer-git --save

11、部署到Coding

执行以下命令将生成的博客静态页面 push 到Coding 创建的仓库中

hexo clean
hexo generate
hexo deploy

开启Coding Pages 服务后即可访问博客，如果有自己的域名也可以绑定自己的域名，方便访问。具体可以参考官方文档。
